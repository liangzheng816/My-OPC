# PMFrame 100-framework Markdown pack

This folder contains **100 English Markdown summary files** generated from the uploaded PMFrame workbook and cross-checked against recognized framework lineages.

## What is included

- The **99 main frameworks** listed in the workbook under:
  - User insights
  - Problem framing
  - Ideation
  - Validation
  - Execution
  - Growth
  - Systems thinking
- **Pre-mortem** as the 100th entry.

## What is intentionally excluded

- **Reverse Thinking** was **not included** in the 100-file pack because the uploaded workbook itself marks it as a probable additional entry that could not be verified on the original PMFrame homepage. To avoid inventing certainty, I excluded it from the final 100.

## How these files were written

Each file follows the same structure:
- one-paragraph summary
- the problem it solves
- canonical origin / lineage
- canonical structure or common working structure
- when to use it
- practical operating guidance
- inputs / outputs / success criteria
- PM example
- strengths, limitations, and common mistakes
- authority trail / references

## Important honesty note

Not all PM frameworks have one universally official source. Some are:
- **canonical frameworks** with a recognized creator or source text
- **widely used practitioner methods** with several variants
- **composite product-management tools** that exist mainly as operational practice

Where a framework does **not** have one single official definition, the summary explicitly says so and uses a careful synthesis instead of pretending there is one canonical template.
