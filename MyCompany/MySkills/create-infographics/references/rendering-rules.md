# Rendering Rules — The Canonical Infographic Style Guide

This is the **single source of truth** for how infographics should look when generated by the `create-infographics` skill. Apply these rules unless the user explicitly overrides.

The style is named **`executive-saas`** — modern AI / SaaS keynote aesthetic. It pairs cleanly with most enterprise audiences (founders, PMs, analysts, knowledge workers) and renders reliably across Gemini, GPT-Image, and other backends.

---

## Layer 1 — Visual Style (always applies)

### Background & Canvas

| Property | Value | Notes |
| --- | --- | --- |
| **Background** | Pure white `#FFFFFF` or very light slate `#FAFAFA` | Never dark mode — only the bottom banner is dark |
| **Card backgrounds** | White or very light slate `#F8FAFC` | Used for grouping zones |
| **Soft washes** | Blue wash `#EFF6FF`, orange wash `#FFF7ED` | Use to tint cards by semantic meaning |

### Strict Palette (this is the whole palette — nothing else)

| Role | Color | Hex |
| --- | --- | --- |
| Primary text | Cool dark gray | `#0F172A` |
| Secondary text | Slate | `#475569` |
| Muted text | Light slate | `#94A3B8` |
| Card border | Cool gray | `#E2E8F0` |
| Row divider | Lighter cool gray | `#F1F5F9` |
| **Primary accent — future / positive / interactive** | **Enterprise blue** | **`#2563EB`** |
| **Secondary accent — burden / transition / current pain** | **Warm orange** | **`#F97316`** |
| Optional neutral data accent | Muted teal | `#0EA5E9` |
| Soft blue wash | Light blue | `#EFF6FF` |
| Soft orange wash | Light orange | `#FFF7ED` |
| Bottom banner background | Dark slate | `#0F172A` |
| Bottom banner text | White | `#FFFFFF` |

**No other colors.** No rainbows. No tinted gradients on cards. Optional teal `#0EA5E9` only when you need a third neutral data accent.

### Typography

| Element | Treatment |
| --- | --- |
| **Master title** | Large, bold, dark slate `#0F172A`, clean sans-serif (Inter / Helvetica / SF Pro feel) |
| **Subtitle** | Medium weight, slate `#475569` |
| **Tiny attribution** | Small, muted slate `#94A3B8` (source, page, timestamp) |
| **Section headers** | Bold, dark slate, optionally with circular blue numbered badge |
| **Section mini-headers** | Small uppercase tracked labels in slate (e.g., "WHAT DOESN'T CHANGE") |
| **Body text** | Medium weight, slate `#475569`, comfortable line-height |
| **Callouts / quotes** | Larger weight, dark slate, with subtle italic or accent color emphasis |
| **Numbers / metrics** | Slightly larger than body, in accent color (blue or orange depending on meaning) |

### Cards & Spacing

- **Rounded corners**: 8-12px radius
- **Drop shadows**: subtle, ~6-10% opacity black, small Y-offset
- **Borders**: thin 1px cool gray `#E2E8F0`
- **Padding**: generous — let cards breathe
- **Gaps**: equal horizontal/vertical spacing between cards
- **Alignment**: card baselines aligned across rows

### Icons

- **Simple line icons** (single-stroke), small and functional
- OR filled icons in the card's accent color
- Avoid stock-vector "corporate art" mascots, clipart, or cartoon characters

### Don'ts (universal)

- ❌ Dark backgrounds (except the bottom banner)
- ❌ Gradients on cards (only the transformation arrow in the 4-stage template uses one)
- ❌ Glossy effects, photorealism, 3D shadows
- ❌ Hand-drawn / chalk / sketch aesthetic (that's the `chalkboard` override)
- ❌ Rainbow palettes (strictly blue + orange + neutrals + optional teal)
- ❌ Stock-clipart mascots, cartoon characters, oversized icons
- ❌ Dense-modules cramming (this is executive keynote, not infopedia poster — unless explicitly chosen)

## Layer 2 — Standard Structural Zones

Every infographic uses **the same 4 zones**, in this top-to-bottom order:

```
┌──────────────────────────────────────────────────┐
│  ZONE 1 — Top header strip                       │
│  Master title · Subtitle · Tiny attribution      │
├──────────────────────────────────────────────────┤
│  ZONE 2 — Hero / thesis band                     │
│  Full-width callout card (light blue wash)       │
│  Single bold statement or master equation        │
├──────────────────────────────────────────────────┤
│  ZONE 3 — Main content                           │
│  (layout-specific — varies by content shape)     │
│                                                  │
│  Examples:                                       │
│    - 3 cards horizontally                        │
│    - 4-stage maturity arc                        │
│    - 2-card binary comparison                    │
│    - 6 numbered sections in dense-modules        │
├──────────────────────────────────────────────────┤
│  ZONE 4 — Bottom takeaway banner                 │
│  Full-width, dark slate #0F172A, white text     │
│  Single sentence + smaller attribution line      │
└──────────────────────────────────────────────────┘
```

This 4-zone structure scans **top-to-bottom** in 5-10 seconds, the timing executive audiences expect from a keynote slide. Don't break it without reason.

## Layer 3 — Aspect Ratio Selection

| Aspect | Use for | Section count |
| --- | --- | --- |
| **16:9 landscape** (default) | Section infographics, single-topic deep dives, slide-deck pages | 3-7 cards |
| **9:16 portrait** | Long-form posters summarizing whole talks / papers | **Maximum 6 numbered sections** |
| **3:4 portrait** | Dense-modules infopedia posters | 6-7 modules with tight content |
| 1:1 square | Social-media share images | 3-4 cards |

**Critical:** at 9:16 with more than 6 sections, Gemini starts duplicating section badges (the infamous "two 04" bug). See [anti-duplication.md](anti-duplication.md).

## Layer 4 — Semantic Color Assignment

The palette has **two accent colors carrying semantic weight**:

### Orange `#F97316` — Burden / Pain / Current State

Use orange for:

- "How it works today" (problematic)
- Operational pain points
- Failure modes, errors, warnings
- Pre-transformation states
- Stage 1 and 2 of a 4-stage maturity arc
- "Old" half of a binary comparison

### Blue `#2563EB` — Future / Positive / Target State

Use blue for:

- "Where it's heading" (better)
- Aspirational outcomes
- Stage 3 and 4 of a 4-stage maturity arc
- "New" half of a binary comparison
- Default for everything when there's no semantic dichotomy

### Transformation Arrow

When a slide has a clear "old → new" arc, the connecting arrow uses an **orange → blue gradient**. This is the only place gradients are allowed.

### Optional Teal `#0EA5E9`

Use sparingly as a third neutral accent when:

- A slide has 3 categorically different cards (orange + blue + teal)
- A neutral data accent is needed (e.g., "dimension" data type alongside "material" and "supplier")

## Layer 5 — Layout Cookbook (Quick Reference)

The most-used layouts in this skill:

### A. 4-stage Maturity Arc (16:9 landscape) — for transformation content

Apply when content has a clear "current → future" progression:

1. Current deterministic / manual state — orange
2. Human operational burden — orange (deeper wash)
3. Intent-driven AI state — blue (lighter wash)
4. Autonomous execution loop — blue (deepest)

Each stage: numbered circular badge, short heading, simple line icon, 1-2 bullets, micro-metric.
Thin gray arrows between 1→2 and 3→4. Orange→blue gradient arrow between 2→3.

### B. 6-section dense-modules poster (9:16 portrait) — for whole-talk summaries

Master title + thesis band + 6 numbered sections + bottom takeaway. Each section gets ~12% of vertical space. Mix of comparison cards, lists, mini-diagrams within sections. **Maximum 6 sections — see anti-duplication.md.**

### C. Binary comparison (16:9 landscape) — for A vs B

Two side-by-side card halves. LEFT in orange (current / old / pain), RIGHT in blue (future / new / target). Optional transformation arrow between with orange→blue gradient. Bottom band with shared takeaway.

### D. 3-card horizontal (16:9 landscape) — for parallel themes

Three equal-width cards. If themes are categorically different, color-code (blue / orange / teal). If they're peers, all blue. Numbered circular badges at top of each card.

### E. Single hero stat with supporting band (16:9 landscape) — for one big number

One oversized number/metric centered in the hero band. Three smaller supporting metrics in cards below. Used for single-stat amplification (e.g., "67% organizational vs 32% individual").

## Layer 6 — Content Quality Bar

Beyond visual style, content quality matters:

### What goes in the slide

- **Master title**: 3-7 words, declarative
- **Subtitle**: one short sentence
- **Section headers**: 2-4 words
- **Bullets**: 6-12 words max each
- **Numbers**: bold, in accent color
- **Quotes**: short (under 20 words), attributed
- **Bottom takeaway**: one sentence that summarizes the slide's argument

### What stays out

- Long paragraphs (split into bullets or move to body text)
- Disclaimers, methodology footnotes (use the tiny attribution line)
- Multiple competing arguments (pick one per slide)
- Hashtags or social-media language
- Brand watermarks beyond the tiny attribution

## Layer 7 — When To Override

The user can override the default style with explicit instruction. Common overrides:

| User says | Switch to |
| --- | --- |
| "Use chalkboard" or "classroom feel" | `chalkboard` — dark green-black bg, chalk fonts, hand-drawn |
| "Make it academic" or "research paper feel" | `aged-academia` — sepia parchment, ink illustrations |
| "Make it technical" or "engineering blueprint" | `technical-schematic` — blueprint blue, dimension lines |
| "Make it warm" or "hand-drawn" | `morandi-journal` — Morandi tones, doodles |
| "Make it lab-precision" or "1970s-pop-tech" | `pop-laboratory` — blueprint grid + pop accents |
| "Make it data-dense" or "dense-modules" | `dense-modules` layout with executive-saas style |

When overriding, see [`deps/baoyu-infographic/references/styles/`](../deps/baoyu-infographic/references/styles/) for the full style definitions.

**Override always wins.** Don't second-guess the user's explicit choice.

## Layer 8 — The Anti-Duplication Rules (READ THIS)

These rules emerged from real Gemini failures in production. See [anti-duplication.md](anti-duplication.md) for the full set. Summary:

1. **Maximum 6 numbered sections at 9:16** — beyond this, Gemini duplicates badges
2. **State section count and sequence explicitly** in the prompt — e.g., "EXACTLY 6 numbered sections, in this order: 01, 02, 03, 04, 05, 06"
3. **Number cards explicitly** — e.g., "Card 1 of 3", "Pillar 1 of 2"
4. **Add a critical rules block at the top of every prompt** — Gemini reads top-of-prompt instructions more strongly
5. **Spell out math operators** — use "multiplied by" instead of `×` when reliability matters
6. **Never put structure rules at the bottom** — they get ignored when buried
