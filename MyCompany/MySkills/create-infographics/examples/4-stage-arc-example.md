# Example — 4-Stage Maturity Arc (Executive Slide)

A worked example showing the prompt for a 4-stage maturity arc — the canonical narrative template for "current → future" transformation content.

**Source content:** Sonya Huang's "Sliding Scale of Agenticness" from Sequoia AI Ascent 2026.

**Layout chosen:** 4-Stage Maturity Arc (16:9 landscape).

---

## When to Use This Layout

Apply the 4-stage arc when content fits this exact shape:

1. **Current state** — deterministic, manual, or limited
2. **Operational burden** — humans carry the weight
3. **Intent-driven** — AI handles execution, humans set direction
4. **Autonomous loop** — fully autonomous with guardrails

This template fits a lot of agent / AI-transformation narratives. If content doesn't have a 4-stage arc, use a different layout (binary, n-card, or dense-modules poster).

---

## The Prompt

```text
Create a 16:9 executive infographic slide showing the evolution from "TAB AUTOCOMPLETE (2023)" to "AUTONOMOUS DARK FACTORIES (2026)".

Use four horizontally arranged maturity stages — Sonya Huang's Sliding Scale of Agenticness:

1. Current deterministic / manual state — "TAB AUTOCOMPLETE" — One AI assisting a human in-line. 2023 baseline. Incrementally useful, fundamentally not transformative. Humans do all reasoning and direction.

2. Human operational burden — "AGENTIC DEV" — One human directs an agent (or a team of agents). Constant context-switching, prompt iteration, output review. Current dominant paradigm. The human is the bottleneck.

3. Intent-driven AI state — "BACKGROUND / ASYNC AGENTS" — Agents run on long horizons in the background, spawn sub-agents, hand off work. Humans set intent and review outcomes — not steps. Likely to overtake current paradigm in volume.

4. Autonomous execution loop — "DARK FACTORIES" — No human review, only guardrails. Self-monitoring, self-improving execution. Already in production at some cybersecurity firms.

Each stage should be shown as a rounded card with:
- A numbered circular badge (1 / 2 / 3 / 4) at the top
- A short heading (the stage name in bold, 2-4 words)
- A simple line icon representing the stage:
  - Stage 1: a keyboard cursor / tab arrow
  - Stage 2: a human figure with a dashboard
  - Stage 3: branching forking arrows
  - Stage 4: a closed self-feeding loop / cycle icon
- Minimal body text (1-2 short bullets per card; ~10 words max each)
- One micro-stat per card:
  - Stage 1: "1 AI / 1 line"
  - Stage 2: "1 human / 1 agent team"
  - Stage 3: "1 human / N async agents"
  - Stage 4: "0 humans in loop"

Use thin arrows between stages (1→2, 2→3, 3→4):
- 1→2 and 3→4 arrows in medium gray (#94A3B8)
- 2→3 arrow uses an orange→blue gradient to mark the transformation moment

Color assignments:
- Stages 1 and 2 use warm orange (#F97316) — current paradigm + human burden
- Stage 1 background: very light orange wash (#FFF7ED)
- Stage 2 background: slightly deeper orange wash
- Stages 3 and 4 use enterprise blue (#2563EB) — future / autonomous
- Stage 3 background: very light blue wash (#EFF6FF)
- Stage 4 background: slightly deeper blue wash

Below the 4 stages, a small horizontal supporting band (subtle gray background #F8FAFC, full width) with three stat callouts side-by-side:

- METR CHART · Sustained progress: tens of min → hours in one year
- AGENT ANATOMY · Brain (Reason+Plan) · Tools (Arms&Legs) · Harness (Persistence)
- 100 → 100 · "100 years possible in 100 days" — Zed · Sierra · Notion

Add a bottom strategic takeaway banner in dark slate (#0F172A) with white text, full width:

"Build agent harnesses, not autocomplete. Product builders must transition toward intent-driven, goal-state design."

Master title at top (large, bold, dark slate #0F172A):
"Sonya Huang — State of Agents 2026"

Subtitle (medium weight, slate #475569):
"The sliding scale of agenticness — from in-line assistance to autonomous loops"

## Visual style — modern enterprise AI / SaaS keynote

- Background: pure white (#FFFFFF) or off-white (#FAFAFA)
- Typography: clean modern sans-serif (Inter / Helvetica feel)
- Whitespace: generous
- Cards: rounded rectangles, subtle drop-shadows, thin 1px cool-gray (#E2E8F0) borders
- Icons: simple line icons or filled in the stage's accent color
- Equal horizontal gaps; aligned baselines

Strict palette:
- White / off-white background
- Primary text #0F172A
- Secondary text #475569
- Card border #E2E8F0
- Blue #2563EB (future)
- Orange #F97316 (burden)
- Optional neutral: teal #0EA5E9

Don'ts:
- No dark backgrounds, gradients on cards, glossy effects, photorealism
- No hand-drawn, sketchy, chalk aesthetic
- No rainbow palettes
```

---

## Key Lessons This Example Demonstrates

1. **4-stage arc is a narrative template** — fits content with a clear "current → future" progression
2. **Semantic color split is intentional** — orange for stages 1-2 (current pain / human burden), blue for stages 3-4 (future / autonomous)
3. **The 2→3 arrow is the transformation moment** — only place gradients are allowed (orange → blue)
4. **Micro-stat per card** — small but vivid (e.g., "1 AI / 1 line", "0 humans in loop")
5. **Supporting band below the 4 cards** — for related but not-on-the-arc data (METR chart, agent anatomy)
6. **Bottom takeaway formula** — "Build [FUTURE METAPHOR], not [OLD METAPHOR]. Product builders must transition toward [NEW DESIGN PRINCIPLE]."
7. **No anti-duplication block needed** — 4 stages with distinct titles is below Gemini's failure threshold

---

## Pattern Recognition

This layout works well for:

- "Adoption to absorption" frameworks (any 4-stage maturity model)
- "Generations" or "waves" content (Gen 1 → Gen 2 → Gen 3 → Gen 4)
- Capability evolution (pre-AI → ChatGPT → reasoning → agents)
- Workflow maturity (manual → assisted → augmented → autonomous)
- Sliding scales (any 4-step progression)

It does NOT fit:

- Parallel themes (use n-card horizontal)
- A vs B comparisons (use binary)
- Single hero stat (use hero stat + supporting)
- Whole-talk summary (use 6-section dense-modules poster)
