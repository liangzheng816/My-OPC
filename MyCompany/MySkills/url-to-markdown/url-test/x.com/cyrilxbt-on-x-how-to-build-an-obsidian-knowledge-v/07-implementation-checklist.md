# Implementation Checklist

> A six-step starting checklist plus the "start with five notes tonight" entry-point that defuses setup overwhelm — the most common reason people never build the system.

## Key Points
- **Step 01** — Install Obsidian and create the **five-folder structure** (Inbox, Notes, Ideas, Projects, plus `CLAUDE.md` in the root). Do not add other folders. Let structure evolve from actual usage.
- **Step 02** — Connect Readwise to the vault using the **native Obsidian integration**. Every highlight you make anywhere will automatically appear in `/notes` as a formatted markdown file.
- **Step 03** — Build the Telegram capture bot with N8N (workflow shown in the Automated Capture chapter). **30 minutes of setup**; handles every quick capture from your phone for the rest of your life.
- **Step 04** — Write your `CLAUDE.md` from the template. Be specific and honest. **The quality of Claude's output is directly proportional to the quality of context here.**
- **Step 05** — Set up the **daily brief automation** in N8N. Schedule it for **6am every weekday**. Output goes to `/inbox`. Read it before you open anything else.
- **Step 06** — Block **15 minutes every Monday** for the weekly synthesis. Put it in the calendar now. **"Do not skip it in week two because the vault feels too empty."**
- **The 5-note starter**: the most common reason people never build this system is that it feels like too much to set up at once. Today, **put 5 notes in Obsidian**. Anything — five articles you've been meaning to read, five ideas, five questions. Connect Claude to that folder. Ask it to find connections.
- **"It will find something you missed. It always does."** That moment — when Claude surfaces a connection between two things you thought were unrelated — is when the system stops being a concept and becomes something you want to feed every day.

## Details

The 6-step checklist is ordered by **dependency, not by difficulty**. You install Obsidian and create the folders before connecting tools because the folders are the destination. CLAUDE.md goes after the capture-tool wiring because CLAUDE.md describes a working vault, not an empty one.

Steps 01–04 are all **one-time setup** with no ongoing cost. Steps 05–06 establish the two ongoing rituals (automated daily brief, 15-min weekly session) that make the system load-bearing. Without those two, you have automated capture but no return loop — back to the failure mode from chapter 1.

The "five notes tonight" entry-point is the author's acknowledgment that the previous 6 steps look like a weekend project. The smaller ask is structurally identical to the full system in miniature: capture (5 notes) + Claude reading them + finding connections. The author's confidence — "It always does" — is calibrated to the experience: any 5 notes from a thinking person have non-obvious connections, and Claude is good enough at surfacing them that the demo never fails.

The chapter closes with the entry-cost argument addressed to the procrastinator: **"Start with five notes tonight. The vault does the rest."** Five notes is below any reasonable activation-energy threshold; once five notes exist, capture #6 is trivial because the system is already running.
