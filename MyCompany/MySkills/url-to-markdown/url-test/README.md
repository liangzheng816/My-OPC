# url-test — example outputs

End-to-end example outputs from this skill, run against an X (Twitter) thread by [@cyrilXBT](https://x.com/cyrilXBT/status/2052235121416188114) — *"How to Build an Obsidian Knowledge Vault That Gets Smarter Every Day Without You Doing Anything"* — a 2,499-word thread on building a Claude-powered second brain.

These are real artifacts produced by the skill in a Claude Code session. They show what Steps 1, 2a, and 2b produce in practice. **No API key was used at any point** — the LLM steps were performed by the host agent.

## What each path demonstrates

| Path | Skill step | Notes |
|---|---|---|
| `*.md` | Step 1 (deterministic) | Generated by `bun scripts/main.ts <url>`. Auto-mode escalated to headless Chrome because the X SPA returned a JavaScript-required shell. No API key required. |
| `*.summary.md` | Step 2a (agent) | Host agent applied [`prompts/summary.md`](../prompts/summary.md) to the extracted article body. |
| `<slug>/00-index.md` + `NN-*.md` | Step 2b (agent) | Outline pass → 7 sections derived from the source's natural arc → parallel per-chapter generation → deterministic index. |

The **per-chapter MDs** are formatted as direct input for downstream chapter-infographic generation (each has `# Title`, framing blockquote, `## Key Points`, `## Details`, `## Visuals`).

## Reproduce

```bash
SKILL=~/.claude/skills/url-to-markdown/scripts/main.ts

# Step 1
npx -y bun "$SKILL" \
    "https://x.com/cyrilXBT/status/2052235121416188114" \
    --out ./url-test
```

Step 2a / 2b are performed by the host agent in a Claude Code / Codex session — paste the URL with intent ("give me a summary" → 2a; "break this into chapter notes" → 2b) and the agent will follow the workflow in [SKILL.md](../SKILL.md).

## What this example shows about the skill

- **Auto-escalation works**: plain `fetch` matched the bot-block marker `Please enable JavaScript`; auto-mode launched Chrome and produced clean output without any flags.
- **Output quality survives Chrome rendering**: Readability + Turndown produced clean prose with no Twitter sidebar / replies / "follow this account" widgets — same quality as the fetch path, just sourced from rendered HTML.
- **Sections-mode handles articles without explicit Markdown headings**: this thread used ALL-CAPS section breaks (e.g., `WHY MOST KNOWLEDGE SYSTEMS FAIL`) rather than `##` headers. Pass A's `ANCHOR:` field still located each section reliably.
